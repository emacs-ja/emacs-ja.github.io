2.2 Visibility Cycling💡ビジビリティ・サイクリング

Outlines make it possible to hide parts of the text in the buffer. Org uses just two commands, bound to TAB and {{{kbd{S-TAB)}}} to change the visibility in the buffer.
💡アウトラインは、バッファ内のテキストの一部を隠すことを可能にします。Orgは、バッファ内の可視性を変更するために、TAB と S-TAB にバインドされたたった2つのコマンドを使用します。

TAB

Subtree cycling: Rotate current subtree among the states
💡サブツリーの循環：現在のサブツリーを状態間で回転させる 

,-> FOLDED -> CHILDREN -> SUBTREE --.
'-----------------------------------'

When called with a prefix argument (C-u TAB), or with the Shift key, global cycling is invoked.
💡プレフィックス引数（C-u TAB）、またはShiftキーで呼び出された場合、グローバルサイクリングが起動される。

S-TAB
C-u TAB

Global cycling: Rotate the entire buffer among the states
💡グローバルサイクリング。バッファ全体を状態間で回転させる 

,-> OVERVIEW -> CONTENTS -> SHOW ALL --.
'--------------------------------------'
C-u C-u C-u TAB

Show all, including drawers.
💡ドロワーも含めてすべて表示する。

When Emacs first visits an Org file, the global state is set to OVERVIEW, i.e., only the top level headlines are visible. This can be configured through the variable org-startup-folded, or on a per-file basis by adding a ‘STARTUP’ keyword to ‘overview’, ‘content’, ‘showall’, ‘showeverything’ or ‘show<n>levels’ (n = 2..5) like this:
💡Emacsが最初にOrgファイルを訪れたとき、グローバルな状態はOVERVIEWに設定されます、つまり、トップレベルのヘッドラインのみが表示 されます。これはorg-startup-foldedという変数で設定できます．また，ファイルごとに，'overview', 'content', 'showall', 'showeverything', 'show<n>levels' (n = 2..5) に以下のような 'STARTUP' キーワードを追加して設定することも可能です．

#+STARTUP: content
